1. 简介

1.1 什么是GeoSHAP？

GeoSHAP是一个专业的地理空间数据分析和SHAP值解释的综合性工具。它集成了机器学习、空间统计分析、模型解释于一体，为地理空间数据分析提供了一个完整、专业的解决方案。

1.2 主要特点

  • ✅ 零代码操作：图形化界面，无需编程即可完成复杂分析
  • ✅ 双任务支持：支持回归和分类两种问题类型
  • ✅ 模型丰富：集成XGBoost、LightGBM、Random Forest等主流算法
  • ✅ 空间分析：Moran's I、LISA等空间统计方法
  • ✅ 模型解释：SHAP值分析，让AI决策过程透明化
  • ✅ 一键报告：自动生成HTML/PDF格式的专业报告
  • ✅ 性能优化：GPU加速、内存优化、并行计算支持
  • ✅ 项目管理：支持项目保存和加载，方便工作流程管理
  • ✅ 多语言支持：支持中英文界面切换
  • ✅ 主题切换：支持亮色和暗色主题

1.3 适用场景

  • 🌍 环境监测：空气质量预测、土壤污染评估、水质分析
  • 🏙️ 城市研究：房价预测、交通流量分析、城市热岛效应
  • 🌾 农业遥感：作物产量预测、病虫害风险评估、土壤适宜性分析
  • 🚨 灾害评估：洪水风险评估、地震危险性分析、滑坡易发性预测
  • ⛏️ 资源勘探：矿产资源预测、地下水分布预测、能源资源评估
  • 🗺️ 土地利用：土地分类、土地利用变化监测、土地适宜性评价

2. 系统要求与安装

2.1 系统要求

操作系统

  • Windows: Windows 7/10/11 (64位)
  • Linux: Ubuntu 18.04+ 或其他主流Linux发行版
  • macOS: macOS 10.14+ (Mojave及以上)

硬件要求

  • CPU: 双核及以上（推荐四核或更多）
  • 内存: 8GB以上（处理大数据集建议16GB+）
  • 硬盘: 至少2GB可用空间
  • GPU（可选）: NVIDIA GPU（支持CUDA），用于XGBoost加速

Python版本

  • Python 3.7 - 3.13

2.2 安装步骤

步骤1：创建虚拟环境（推荐）

# 使用conda创建环境（推荐）
conda create -n geoshap python=3.10
conda activate geoshap

# 或使用venv
python -m venv geoshap_env
# Windows
geoshap_env\Scripts\activate
# Linux/macOS
source geoshap_env/bin/activate

步骤2：安装依赖库

pip install ttkbootstrap rasterio geopandas pandas numpy scikit-learn shap matplotlib joblib statsmodels xgboost lightgbm catboost scipy seaborn scikit-optimize pysal esda libpysal jinja2 weasyprint lime tqdm

注意：某些库可能需要额外的系统依赖：

  • rasterio: 需要GDAL库
    • Windows: 下载GDAL wheel文件或使用conda安装
    • Linux: sudo apt-get install gdal-bin libgdal-dev
    • macOS: brew install gdal
  • geopandas: 需要GEOS、PROJ等空间库
    • 通常随geopandas一起安装
  • XGBoost GPU支持: 需要CUDA Toolkit和cuDNN

步骤3：验证安装

python -c "import geopandas, rasterio, shap, xgboost; print('所有依赖安装成功！')"

步骤4：运行程序

python GeoShapv4.5.0.py

2.3 GPU加速配置（可选）

如果需要使用GPU加速XGBoost训练：

  1. 安装CUDA Toolkit

    • 下载并安装NVIDIA CUDA Toolkit（版本11.0+）

  2. 安装支持CUDA的XGBoost

3. 快速开始

3.1 5分钟快速上手

步骤1：准备数据

栅格数据准备

  • 将所有预测变量（特征）的栅格文件放在同一个文件夹中
  • 支持的格式：GeoTIFF（.tif）、其他GDAL支持的格式
  • 要求：
    • 所有栅格必须具有相同的投影坐标系（CRS）
    • 所有栅格必须具有相同的空间分辨率
    • 所有栅格必须覆盖相同的空间范围
    • 栅格文件名将作为特征名称使用

样本数据准备

  • 使用矢量文件（Shapefile）存储样本点的位置和属性
  • 支持的格式：Shapefile（.shp）、GeoJSON、GPKG等
  • 要求：
    • 必须包含几何信息（点、面都可以）
    • 必须包含目标变量（要预测的变量）列
    • 坐标系应与栅格数据一致（不一致时程序会自动转换）

步骤2：启动程序并加载数据

  1. 启动GeoSHAP程序
  2. 切换到"📁 Input Data"标签页
  3. 选择问题类型（回归/分类）
  4. 点击"Browse"选择栅格文件夹
  5. 点击"Browse"选择样本矢量文件
  6. 选择目标变量列（或点击"🔍 Detect"自动检测）
  7. 设置输出文件夹路径
  8. 点击"🔄 Refresh Preview"查看数据预览

步骤3：配置模型参数

  1. 切换到"⚙️ Options"标签页
  2. 选择要训练的模型（建议先选择1-2个模型测试）
  3. 保持其他参数为默认值
  4. 点击"🚀 Start Analysis"按钮

步骤4：查看结果

  1. 等待训练完成（查看"📋 Log"标签页的进度）
  2. 切换到"📈 Model Evaluation"标签页查看模型性能
  3. 切换到"📊 SHAP Analysis"标签页查看特征重要性
  4. 切换到"📄 Report"标签页生成完整报告

3.2 使用示例数据快速测试

如果您想快速了解程序功能，可以使用内置的样本数据生成器：

  1. 切换到"📁 Input Data"标签页
  2. 在"Generate Sample Data"区域设置参数：
    • Samples: 500（样本数量）
    • Features: 8（特征数量）
    • Problem Type: regression（问题类型）
    • Add spatial pattern: ✓（添加空间模式）
  3. 点击"🎲 Generate"生成数据
  4. 生成的数据会自动加载
  5. 按照步骤3和4进行模型训练和结果查看

5. 详细功能说明

5.1 数据准备

栅格数据要求

  1. 格式要求

    • 推荐使用GeoTIFF格式（.tif）
    • 支持所有GDAL支持的栅格格式（.tif, .img, .hdf等）

  2. 空间要求

    • 所有栅格必须具有相同的投影坐标系（CRS）
    • 所有栅格必须具有相同的空间分辨率
    • 所有栅格必须覆盖相同的空间范围（建议完全重叠）
    • 如果空间范围不完全一致，程序会使用交集区域

  3. 命名建议

    • 使用描述性的文件名（如：elevation.tif, slope.tif, ndvi.tif）
    • 文件名将作为特征名称使用
    • 避免使用特殊字符和空格

  4. 数据质量

    • 确保数据无异常值或已进行预处理
    • 检查NoData值的处理方式
    • 确保数据类型合理（浮点型推荐）

样本数据要求

  1. 格式要求

    • Shapefile（.shp）：最常用，需要完整的.shp/.shx/.dbf文件
    • GeoJSON（.geojson）
    • GPKG（.gpkg）

  2. 几何类型

    • 点数据：每个点代表一个采样位置
    • 面数据：每个面代表一个区域，目标变量可以是区域平均值或其他统计量
    • 程序会自动提取点位置或面中心点对应的栅格值

  3. 属性要求

    • 必须包含目标变量列（要预测的变量）
    • 目标变量列名不能包含特殊字符
    • 确保目标变量值有效（无异常值、无缺失值）
    • 对于分类任务，确保类别标签为整数（0, 1, 2...）

  4. 坐标系

    • 建议与栅格数据使用相同的坐标系
    • 如果不同，程序会自动转换（但可能影响精度）
    • 建议预先统一坐标系以提高效率和精度

  5. 样本数量

    • 回归任务：建议至少100个样本
    • 分类任务：建议每个类别至少30个样本
    • 样本数量越多，模型性能通常越好

5.2 问题类型选择

回归任务（Regression）

适用场景

  • 预测连续数值
  • 例如：房价、温度、产量、浓度、海拔等

评估指标

  • R² (决定系数): 衡量模型解释的方差比例，越接近1越好
  • RMSE (均方根误差): 预测误差的标准差，越小越好
  • MAE (平均绝对误差): 预测误差的平均值，越小越好
  • MAPE (平均绝对百分比误差): 相对误差，越小越好

可视化图表

  • 预测散点图：预测值 vs 真实值，理想情况下应呈45度直线
  • 残差图：残差 vs 预测值，理想情况下应随机分布
  • 残差分布图：检查残差是否正态分布
  • 学习曲线：训练集和验证集得分随样本数变化
  • 模型对比图：比较不同模型的性能

分类任务（Classification）

适用场景

  • 预测类别标签
  • 例如：土地类型、灾害等级、作物类型、植被类型等

支持类型

  • 二分类: 两个类别（如：滑坡/非滑坡、有/无）
  • 多分类: 多个类别（如：森林/草地/农田/水体、1级/2级/3级/4级）

评估指标

  • Accuracy (准确率): 正确预测的比例
  • Precision (精确率): 预测为正例中实际为正例的比例
  • Recall (召回率): 实际正例中被正确预测的比例
  • F1-Score: 精确率和召回率的调和平均
  • AUC (ROC曲线下面积): 二分类任务的综合评估指标

可视化图表

  • 混淆矩阵：各类别的预测准确情况
  • ROC曲线：二分类任务的性能曲线
  • 分类报告：详细的分类指标（每类别的精确率、召回率、F1）

5.3 模型选择

集成学习算法

Random Forest（随机森林）

  • 优点：训练快、不易过拟合、可解释性好、对缺失值鲁棒
  • 适用：中小型数据集、需要快速结果、特征重要性分析
  • 参数：n_estimators（树的数量，默认200）、max_depth（最大深度，默认None）

Extra Trees（极端随机树）

  • 优点：训练更快、进一步降低过拟合
  • 适用：与Random Forest类似，但训练更快
  • 参数：与Random Forest类似

Gradient Boosting（梯度提升）

  • 优点：通常性能较好、可处理非线性关系
  • 适用：中小型数据集
  • 参数：n_estimators、learning_rate（学习率，默认0.1）、max_depth

Histogram-based Gradient Boosting

  • 优点：内存占用小、训练快、适合大数据集
  • 适用：大数据集、内存受限环境
  • 参数：max_iter（迭代次数）、max_depth、learning_rate

梯度提升框架

XGBoost

  • 优点：
    • 性能优异，通常是最好的选择
    • 支持GPU加速（训练速度提升10-100倍）
    • 自动处理缺失值
    • 正则化防止过拟合
    • 内置特征重要性
  • 适用：大多数场景，特别是大数据集
  • 参数：n_estimators、max_depth、learning_rate、subsample（子采样比例）
  • GPU加速：如果检测到GPU，会自动使用GPU加速

LightGBM

  • 优点：
    • 内存占用小，适合大数据集
    • 训练速度快（通常比XGBoost快）
    • 支持类别特征自动处理
    • 内置特征重要性
  • 适用：大数据集、内存受限环境、需要快速训练
  • 参数：n_estimators、max_depth、learning_rate、num_leaves（叶子节点数）

CatBoost

  • 优点：
    • 专门优化类别特征处理
    • 默认参数表现优异，调参成本低
    • 内置过拟合检测机制
    • 对类别特征自动编码
  • 适用：包含类别特征的数据集、需要快速上手
  • 参数：iterations（迭代次数）、depth（树深度）、learning_rate

模型选择建议

回归任务

  • 数据量大（>10万样本）：优先使用LightGBM或XGBoost
  • 数据量中等（1万-10万）：XGBoost、LightGBM、Random Forest
  • 数据量小（<1万）：Random Forest、Gradient Boosting
  • 需要快速结果：Random Forest（训练快）
  • 有GPU可用：XGBoost（GPU加速）
  • 内存受限：HistGradientBoosting、LightGBM

分类任务

  • 类别不平衡：使用CatBoost或调整class_weight参数
  • 多分类问题：CatBoost、XGBoost表现通常较好
  • 二分类问题：所有模型都可以尝试
  • 包含类别特征：CatBoost优先
  • 需要快速结果：Random Forest

首次使用建议

  • 先选择1-2个模型测试（如XGBoost和Random Forest）
  • 根据结果再决定是否尝试其他模型
  • 避免同时选择所有模型（训练时间长）

5.4 数据预处理

标准化方法

StandardScaler（Z-score标准化）

  • 公式：(x - mean) / std
  • 优点：标准化到均值0标准差1，适合大多数情况
  • 适用：特征分布接近正态分布
  • 推荐：默认选择

MinMaxScaler（最小-最大缩放）

  • 公式：(x - min) / (max - min)
  • 优点：缩放到[0,1]区间，易于理解
  • 适用：需要固定范围的特征
  • 注意：对异常值敏感

RobustScaler（鲁棒缩放）

  • 使用中位数和四分位距（IQR）
  • 优点：对异常值鲁棒
  • 适用：数据包含异常值
  • 注意：不保证均值0标准差1

Normalizer（L2归一化）

  • 按L2范数归一化
  • 适用：需要单位向量的场景（较少使用）

MaxAbsScaler（最大绝对值缩放）

  • 按最大绝对值缩放
  • 适用：稀疏数据

注意：树模型（Random Forest、XGBoost、LightGBM）通常不需要标准化，但标准化不会有害。线性模型（如果使用）需要标准化。

特征工程

Interaction（交互特征）

  • 自动创建特征对的乘积和比值
  • 捕捉特征间的相互作用
  • 注意：可能产生大量特征（特征数平方级增长）
  • 适用：特征数量较少时（<20）

Statistical（统计特征）

  • 计算每行数据的统计量：均值、标准差、最大值、最小值、中位数、范围
  • 适用：特征较多时，捕捉样本的整体特征水平
  • 注意：可能丢失空间信息

Ratio（比率特征）

  • 创建特征间的比值
  • 捕捉相对关系
  • 适用：特征间存在比例关系时

Log Transform（对数变换）

  • log(x+1)变换
  • 适用：右偏分布的特征
  • 作用：使分布更接近正态分布
  • 注意：只能应用于正值

Power（幂次变换）

  • x^p变换，可尝试不同幂次
  • 适用：需要非线性变换的特征
  • 注意：可能产生异常值

Binning（分箱）

  • 将连续特征离散化为类别特征
  • 捕捉非线性关系
  • 注意：可能丢失信息，需要合理设置分箱数

Clustering（聚类特征）

  • 基于特征值对样本聚类，将聚类标签作为新特征
  • 捕捉样本的相似性模式
  • 适用：样本间存在明显聚类模式时

Polynomial（多项式特征）

  • 创建多项式特征（如x², x³, x*y）
  • 捕捉非线性关系
  • 注意：可能产生大量特征，导致过拟合
  • 适用：特征数量少、样本数量多时

使用建议

  • 首次运行：不启用特征工程，使用原始特征
  • 性能不佳时：尝试启用特征工程
  • 选择方法：根据数据特点选择，避免使用所有方法（可能导致过拟合）
  • 特征数量控制：启用特征工程后，注意特征数量增长，可能需要启用特征选择

特征选择

相关性过滤

  • 移除相关性超过阈值的特征对
  • 阈值建议：0.90-0.95
  • 作用：减少多重共线性
  • 方法：保留相关性较低的特征

VIF检测

  • 方差膨胀因子（Variance Inflation Factor）
  • 阈值建议：10-15
  • 作用：识别多重共线性问题
  • 方法：移除VIF值超过阈值的特征

使用建议

  • 特征数量多（>50）：建议启用特征选择
  • 特征数量少（<20）：可以不启用
  • 阈值设置：根据实际情况调整，避免过度过滤
  • 注意：特征选择可能移除重要特征，需要结合专业知识判断

5.5 超参数优化

优化方法

Grid Search（网格搜索）

  • 遍历参数网格的所有组合
  • 优点：全面搜索，能找到最优参数
  • 缺点：耗时，参数空间大时不适用
  • 适用：参数空间小（<1000种组合）
  • 计算成本：O(n_combinations × n_folds × n_samples)

Random Search（随机搜索）

  • 随机采样参数组合
  • 优点：效率高，通常能找到接近最优的参数
  • 缺点：可能错过最优参数
  • 适用：参数空间大，时间有限
  • 计算成本：O(n_iterations × n_folds × n_samples)

Bayesian Optimization（贝叶斯优化）

  • 基于历史评估结果智能选择下一组参数
  • 优点：最省时高效，通常能找到最优参数
  • 缺点：需要安装scikit-optimize库
  • 适用：参数空间大，时间有限，追求最优性能
  • 计算成本：O(n_iterations × n_folds × n_samples)，但通常n_iterations更小

优化策略

首次运行

  • 不启用超参数优化
  • 使用默认参数，快速了解数据特征
  • 评估模型性能基线

正式分析

  • 选择表现最好的2-3个模型进行优化
  • 使用Random Search或Bayesian Optimization
  • 设置合理的迭代次数（20-50次）
  • 根据时间预算调整迭代次数

深度优化

  • 使用Grid Search在关键参数上精细搜索
  • 结合交叉验证选择最优参数
  • 注意避免过拟合（测试集性能与训练集性能差距过大）
  • 使用空间交叉验证（如果适用）

参数范围设置

  • 根据模型文档和最佳实践设置参数范围
  • 避免范围过大（搜索效率低）
  • 避免范围过小（可能错过最优值）
  • 可以点击"Model Parameters"查看默认参数范围

5.6 空间分析

空间交叉验证

为什么需要空间交叉验证？

  • 空间数据具有自相关性
  • 随机划分会导致训练集和测试集数据相似
  • 造成过于乐观的评估结果
  • 空间交叉验证基于空间位置划分数据，更真实反映模型泛化能力

实现方式

  • 基于空间聚类的CV：使用K-means对样本点聚类，每个簇作为一个fold
  • 基于空间网格的CV：将研究区域划分为网格，每个网格作为一个fold
  • 确保测试集的样本与训练集样本在空间上保持距离

使用方法

  • 在"Options"标签页勾选"Use Spatial CV"
  • 程序会自动使用空间交叉验证
  • 注意：空间CV可能比普通CV的得分低，这是正常的

空间自相关分析

Moran's I（全局空间自相关）

  • 检测整个数据集是否存在空间聚集模式
  • 取值范围：-1 到 1
  • 正值：相似值聚集（正空间自相关）
  • 负值：不同值聚集（负空间自相关）
  • 零值：空间随机分布
  • p值：判断是否显著（通常<0.05认为显著）
  • z值：标准化统计量

LISA（局部空间自相关）

  • 识别局部空间聚集区域
  • HH：高值聚集区（高值周围也是高值）
  • LL：低值聚集区（低值周围也是低值）
  • HL：高值异常（高值被低值包围）
  • LH：低值异常（低值被高值包围）

空间权重矩阵

  • KNN（K最近邻）: 每个样本的K个最近邻
    • 参数：K值（建议5-15）
    • 适用：样本分布不均匀时
  • Queen（Queen邻接）: 共享边或顶点的邻接
    • 适用：面数据或规则网格数据
  • Distance（距离阈值）: 距离小于阈值的样本
    • 参数：距离阈值（需要根据数据范围设置）
    • 适用：样本分布均匀时

使用方法

  • 在"Spatial Analysis"标签页选择变量和权重类型
  • 设置K值或距离阈值
  • 点击"Compute Moran's I"计算
  • 查看结果和图表

结果解释

  • Moran's I > 0 且 p < 0.05：存在显著的正空间自相关
  • Moran's I < 0 且 p < 0.05：存在显著的负空间自相关
  • p >= 0.05：空间自相关不显著（空间随机分布）

5.7 SHAP分析

什么是SHAP？

SHAP（SHapley Additive exPlanations）值基于博弈论，公平分配每个特征对预测的贡献，让模型决策过程完全透明。

SHAP值的性质

  • 可加性：所有特征的SHAP值之和等于预测值与基准值的差
  • 对称性：相同贡献的特征获得相同的SHAP值
  • 虚拟性：对预测无贡献的特征SHAP值为0
  • 效率性：SHAP值完全解释预测结果

SHAP图表类型

beeswarm（蜂群图/摘要图）

  • 展示所有特征的SHAP值分布
  • 颜色表示特征值大小（红色高，蓝色低）
  • 位置表示SHAP值大小（越远离0影响越大）
  • 推荐：首次查看SHAP结果时使用
  • 适用：快速了解特征重要性

bar（条形图）

  • 展示特征的平均重要性（平均|SHAP值|）
  • 按重要性排序
  • 推荐：快速了解特征重要性
  • 适用：特征重要性排序

heatmap（热图）

  • 展示所有样本的所有特征的SHAP值
  • 可以识别样本和特征的模式
  • 适用：样本数较少时（<1000）
  • 注意：样本数多时可能看不清

dependence（依赖图）

  • 展示单个特征如何影响预测
  • 横轴：特征值
  • 纵轴：SHAP值
  • 颜色：交互特征的值
  • 推荐：深入分析重要特征
  • 适用：理解特征的非线性关系

interaction（交互图）

  • 展示两个特征的交互作用
  • 适用：分析特征间的相互作用
  • 注意：计算成本较高

waterfall（瀑布图）

  • 展示单个样本的SHAP值
  • 逐步展示每个特征如何将基础预测值推向最终预测
  • 适用：解释单个样本的预测
  • 注意：需要指定样本索引

force（力图）

  • 直观展示每个特征的贡献方向和大小
  • 适用：向非技术人员解释预测结果
  • 注意：需要指定样本索引

violin（小提琴图）

  • 展示SHAP值的分布
  • 适用：分析特征值的分布对SHAP值的影响

clustering（聚类图）

  • 基于SHAP值对样本聚类
  • 适用：识别样本的模式

使用建议

  1. 首次分析

    • 先看beeswarm图了解整体情况
    • 再看bar图了解特征重要性排序
    • 识别最重要的特征

  2. 深入分析

    • 选择重要特征，查看dependence图
    • 分析特征的非线性关系和阈值效应
    • 理解特征如何影响预测

  3. 样本解释

    • 选择感兴趣的样本，查看waterfall图或force图
    • 理解为什么该样本得到特定预测结果
    • 用于向他人解释模型预测

  4. 交互分析

    • 如果发现特征间可能存在交互，使用interaction图
    • 理解特征组合的影响

SHAP计算性能

计算时间

  • TreeExplainer（树模型）：快速，适用于Random Forest、XGBoost、LightGBM等
  • KernelExplainer（通用）：慢，适用于所有模型
  • 程序会自动选择最合适的Explainer

优化建议

  • 减少计算SHAP值的样本数量（Background Samples）
  • 对于大数据集，使用TreeExplainer
  • 如果计算时间过长，可以只计算重要特征的SHAP值

7.3 空间插值

功能说明

空间插值功能可以将样本点的预测值或SHAP值插值到整个研究区域。

插值方法

IDW（反距离权重插值）

  • 基于距离的加权平均
  • 参数：
    • power（幂次，默认2）：值越大，近邻影响越大
    • n_neighbors（邻域数，默认12）：参与插值的最近邻数量
  • 适用：大多数情况
  • 优点：简单、快速

Nearest（最近邻插值）

  • 使用最近样本点的值
  • 适用：快速插值，不需要平滑结果
  • 优点：最快
  • 缺点：结果不连续

Linear（线性插值）

  • 基于Delaunay三角剖分的线性插值
  • 适用：需要平滑结果
  • 优点：结果连续
  • 缺点：可能产生异常值

Cubic（三次插值）

  • 基于Delaunay三角剖分的三次插值
  • 适用：需要更平滑的结果
  • 优点：结果最平滑
  • 缺点：计算时间较长

使用步骤

  1. 点击菜单"Tools" -> "Spatial Interpolation"
  2. 选择插值方法
  3. 设置参数
  4. 选择输出路径
  5. 点击"Run Interpolation"开始插值

8. 常见问题解答

8.1 数据相关问题

Q1: 无法加载栅格数据？

可能原因：

  • 文件路径不正确
  • 文件格式不支持
  • 文件损坏
  • GDAL库未正确安装

解决方法：

  • 检查文件路径是否正确
  • 确认文件格式是否支持（推荐使用GeoTIFF）
  • 使用QGIS等工具验证文件完整性
  • 检查GDAL库是否正确安装：gdalinfo --version

Q2: 栅格和矢量数据的坐标系不一致？

现象：程序会显示坐标系转换警告

解决方法：

  • 程序会自动转换坐标系
  • 但建议预先统一坐标系以提高效率和精度
  • 可以使用QGIS等工具预先转换
  • 检查转换后的数据是否正确

Q3: 目标变量列找不到？

可能原因：

  • 列名输入错误
  • 列名包含特殊字符
  • 列不存在
  • 列名区分大小写

解决方法：

  • 点击"🔍 Detect"按钮自动检测列名
  • 检查列名是否正确（区分大小写）
  • 确保列名不包含特殊字符
  • 查看数据预览确认列名

Q4: 数据预览显示异常值？

可能原因：

  • 数据本身包含异常值
  • NoData值未正确处理
  • 数据质量问题

解决方法：

  • 检查原始数据
  • 使用QGIS等工具验证数据
  • 处理NoData值
  • 考虑使用RobustScaler标准化

8.2 训练相关问题

Q5: 模型训练很慢？

可能原因：

  • 启用了超参数优化
  • 使用了大量样本计算SHAP值
  • 数据量大
  • 选择了多个模型
  • 特征数量多

解决方法：

  • 首次运行不启用超参数优化
  • 减少SHAP计算的样本数
  • 使用GPU加速（XGBoost）
  • 减少模型数量，先测试单个模型
  • 减少特征数量或启用特征选择

Q6: 内存不足错误？

可能原因：

  • 数据量太大
  • 特征数量太多
  • 启用了特征工程（产生大量特征）
  • 系统内存不足

解决方法：

  • 减少特征数量
  • 使用内存占用小的模型（如HistGradientBoosting）
  • 不启用特征工程
  • 增加系统内存或使用更高配置的计算机
  • 减少样本数量（如果可能）

Q7: 训练被意外中断？

解决方法：

  • 已完成的模型结果会保存
  • 可以重新运行，程序会跳过已完成的模型
  • 建议经常保存项目
  • 检查是否有错误信息

Q8: 模型性能很差？

可能原因：

  • 特征选择不当
  • 数据质量问题
  • 模型选择不当
  • 参数设置不当
  • 样本数量不足
  • 特征与目标变量相关性低

解决方法：

  • 检查数据质量（缺失值、异常值）
  • 尝试不同的模型
  • 启用特征工程
  • 启用超参数优化
  • 检查特征与目标变量的相关性
  • 增加样本数量
  • 使用空间交叉验证评估真实性能

9.2 评估指标说明

回归指标

R² (决定系数)

  • 范围：-∞ 到 1
  • 含义：模型解释的方差比例
  • 解释：越接近1越好，1表示完美预测，0表示模型不优于平均值，负值表示模型很差
  • 公式：R² = 1 - (SS_res / SS_tot)

RMSE (均方根误差)

  • 范围：0 到 +∞
  • 含义：预测误差的标准差
  • 单位：与目标变量相同
  • 解释：越小越好，0表示完美预测
  • 公式：RMSE = sqrt(mean((y_true - y_pred)²))

MAE (平均绝对误差)

  • 范围：0 到 +∞
  • 含义：预测误差的平均值
  • 单位：与目标变量相同
  • 解释：越小越好，0表示完美预测
  • 公式：MAE = mean(|y_true - y_pred|)

MAPE (平均绝对百分比误差)

  • 范围：0 到 +∞
  • 含义：相对误差
  • 单位：百分比
  • 解释：越小越好，0表示完美预测
  • 公式：MAPE = mean(|y_true - y_pred| / |y_true|) × 100%
  • 注意：当y_true为0时可能有问题

分类指标

Accuracy (准确率)

  • 范围：0 到 1
  • 含义：正确预测的比例
  • 解释：越接近1越好，1表示全部正确
  • 公式：Accuracy = (TP + TN) / (TP + TN + FP + FN)

Precision (精确率)

  • 范围：0 到 1
  • 含义：预测为正例中实际为正例的比例
  • 解释：越接近1越好，表示预测为正例的样本中真正为正例的比例高
  • 公式：Precision = TP / (TP + FP)

Recall (召回率)

  • 范围：0 到 1
  • 含义：实际正例中被正确预测的比例
  • 解释：越接近1越好，表示实际正例中被正确识别的比例高
  • 公式：Recall = TP / (TP + FN)

F1-Score

  • 范围：0 到 1
  • 含义：精确率和召回率的调和平均
  • 公式：F1 = 2 × (Precision × Recall) / (Precision + Recall)
  • 解释：越接近1越好，平衡了精确率和召回率

AUC (ROC曲线下面积)

  • 范围：0 到 1
  • 含义：ROC曲线下的面积
  • 解释：越接近1越好，0.5表示随机猜测，1表示完美分类
  • 注意：仅适用于二分类任务

9.5 最佳实践建议

数据准备

  1. 仔细检查数据质量

    • 清理异常值
    • 处理缺失值
    • 确保特征有实际意义
    • 验证数据一致性

  2. 统一数据格式

    • 统一坐标系
    • 统一空间分辨率
    • 统一数据范围
    • 统一数据类型

  3. 特征选择

    • 选择与目标变量相关的特征
    • 避免使用无关特征
    • 注意特征间的相关性
    • 结合专业知识选择特征

模型训练

  1. 先简单后复杂

    • 先使用简单模型（如Random Forest）快速了解数据
    • 逐步尝试更复杂的模型
    • 避免一开始就使用所有高级功能

  2. 使用交叉验证

    • 评估模型稳定性
    • 避免过拟合
    • 使用空间交叉验证（如果适用）

  3. 合理设置参数

    • 首次运行使用默认参数
    • 根据结果调整参数
    • 避免过度调参
    • 注意计算时间成本

结果解释

  1. 不仅关注准确度

    • 理解模型机理
    • 利用SHAP值深入分析
    • 结合专业知识验证结果
    • 检查模型是否合理

  2. 注意空间特性

    • 使用空间交叉验证
    • 分析空间自相关
    • 理解空间模式
    • 考虑空间异质性

模型应用

  1. 充分验证模型

    • 在应用前充分验证
    • 注意预测数据的特征分布是否与训练数据一致
    • 检查预测结果是否合理
    • 与已知数据对比

  2. 定期更新模型

    • 适应数据变化
    • 保持模型性能
    • 收集新样本重新训练
    • 监控模型性能

工作流程

  1. 保存项目

    • 经常保存项目
    • 备份重要结果
    • 记录分析过程

  2. 文档记录

    • 记录数据来源
    • 记录参数设置
    • 记录分析结果
    • 生成完整报告

9.6 术语表

CRS (Coordinate Reference System)

  • 坐标参考系统，定义空间数据的坐标系

SHAP (SHapley Additive exPlanations)

  • 基于博弈论的特征重要性解释方法

Moran's I

  • 全局空间自相关统计量，衡量空间数据的聚集程度

LISA (Local Indicators of Spatial Association)

  • 局部空间自相关指标，识别局部空间聚集区域

Cross-Validation (CV)

  • 交叉验证，评估模型泛化能力的方法

Spatial Cross-Validation

  • 空间交叉验证，考虑空间自相关性的交叉验证方法

VIF (Variance Inflation Factor)

  • 方差膨胀因子，衡量多重共线性的指标

IDW (Inverse Distance Weighting)

  • 反距离权重插值，基于距离的空间插值方法

GeoTIFF

  • 包含地理坐标信息的TIFF格式栅格文件

Shapefile

  • ESRI开发的矢量数据格式，包含.shp、.shx、.dbf等文件
